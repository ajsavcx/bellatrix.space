## 2. AI Risk Trend Analysis (График)

**API Endpoint:** `GET /api/analytics/{norad_id}`  
**Источник кода:** `analytics_engine.py → generate_risk_trend()`

Этот график показывает, как **менялся риск-скор** конкретного спутника за последние 7 дней.

### Алгоритм расчёта:

#### Шаг 1: Fingerprint спутника (Seed)
```python
seed = int(norad_id) % 100
np.random.seed(seed)
```
Каждый NORAD ID даёт уникальный "отпечаток" (`seed`). Это значит:
- Для **МКС (25544)**: `seed = 44`
- Для **Hubble (20580)**: `seed = 80`

Этот механизм обеспечивает **стабильность**: один и тот же спутник всегда покажет один и тот же базовый характер риска, не меняясь случайно каждую минуту.

#### Шаг 2: Базовый риск
```python
base_risk = np.random.uniform(10, 50)  # Число от 10 до 50
```
База задаётся один раз и зависит от seed. Обозначает "спокойную" фоновую опасность орбиты данного спутника.

#### Шаг 3: Дневные флуктуации (Volatility)
```python
daily_variation = np.random.normal(0, 5)
score = base_risk + daily_variation + (i * 0.5)  # i = день (0...6)
```
Каждый день к базовому риску добавляется:
- **Случайная флуктуация** `±5` (σ=5) — имитация случайных сближений
- **Линейный тренд** `+0.5` за каждый день — имитация постепенного роста плотности обломков

Итоговый `risk_score` ограничен диапазоном **0–100**.

**Что показывает график:**
- **X-ось:** Дата за последние 7 дней
- **Y-ось:** Risk Score от 0 до 100
- **Восходящий тренд:** Орбита становится опаснее
- **Плоский тренд:** Стабильная ситуация

## 3. Индекс Стабильности (Stability Index)

```python
stability = 100 - (np.std([p.risk_score for p in trend]) * 5)
```

**Что это такое:**  
Stability Index измеряет, насколько **предсказуемой** была орбита за 7 дней.

**Логика расчёта:**
- Считается стандартное отклонение (`std_dev`) всех 7 риск-скоров
- Если скоры "прыгают" сильно (`std_dev` высокий) → стабильность низкая
- Если скоры ровные (`std_dev` низкий) → стабильность высокая

| Stability Index | Значение |
|---|---|
| **80–100** | Высокая стабильность. Орбита предсказуема, манёвры не нужны. |
| **50–79** | Умеренная стабильность. Стоит следить за обновлениями. |
| **0–49** | Низкая стабильность. Рекомендуется маневрирование. |

```python
avg_altitude = round(np.random.uniform(400, 800), 1)
```

Показывает среднюю высоту над уровнем моря (в км) за период мониторинга.  
В текущей версии MVP — это репрезентативное значение для орбитального класса спутника (LEO: 400–800 км).

**Орбитальные зоны:**
- 🟢 **160–600 км** — Низкая Земная Орбита (LEO). Здесь живут МКС, Starlink, Hubble.
- 🟡 **600–2000 км** — Высокий LEO. Радиационные пояса Ван Аллена.
- 🔵 **2000–35786 км** — Средняя орбита (MEO). Зона GPS.
- 🟣 **~35786 км** — Геостационарная орбита (GEO). WeatherSats, TV Broadcast.

## 5. Conjunction Risk (Риск сближения)

```python
conjunctions = ["Minimal", "Elevated", "Nominal", "Potential Warning"]
conjunction_risk = conjunctions[seed % len(conjunctions)]
```

Показывает качественную оценку риска **сближения с другим объектом**.

| Уровень | Описание |
|---|---|
| **Minimal** | Вероятность опасного сближения крайне мала. Объект находится в малонаселённой орбитальной зоне. |
| **Nominal** | Стандартный, ожидаемый уровень риска. Ситуация штатная. |
| **Elevated** | Повышенная плотность объектов на данной орбите или прогнозируется сближение. |
| **Potential Warning** | Зафиксировано потенциально опасное сближение. Рекомендуется мониторинг. |

## 6. Orbital Decay (мм/год)

```python
orbital_decay = round(np.random.uniform(0.1, 2.5), 2)
```

**Что это:** Скорость снижения орбиты под воздействием атмосферного сопротивления (drag).

**Почему важно:**  
Атмосфера Земли существует даже на высоте 400–600 км (очень разрежённая). Она тормозит спутник, постепенно снижая его орбиту. Если не корректировать, спутник сойдёт с орбиты и сгорит в атмосфере.

| Decay Rate | Интерпретация |
|---|---|
| **< 0.5 мм/год** | Минимальный decay. Высокая орбита, низкое торможение. |
| **0.5–1.5 мм/год** | Нормальный для LEO спутников. |
| **> 1.5 мм/год** | Значительный decay. Нужна корректировка орбиты в ближайшее время. |

## 7. Solar Activity Impact (Влияние Солнечной Активности)

```python
solar_impacts = ["Low - Ionospheric Stability", "Moderate - Atmospheric Drag", "High - Geomagnetic Storm"]
solar_activity_impact = solar_impacts[seed % len(solar_impacts)]
```

Солнечная активность напрямую влияет на плотность верхних слоёв атмосферы:

| Уровень | Что происходит |
|---|---|
| **Low — Ionospheric Stability** | Солнце спокойно. Атмосфера плотная и предсказуемая. |
| **Moderate — Atmospheric Drag** | Умеренные вспышки. Небольшое увеличение плотности атмосферы → рост drag на спутники в LEO. |
| **High — Geomagnetic Storm** | Сильная вспышка / корональный выброс. Атмосфера резко "раздувается". Drag возрастает в 5–10 раз. Орбиты снижаются быстро. |

> В феврале 2026 году мощная геомагнитная буря уничтожила 40 спутников Starlink, когда повышенное сопротивление атмосферы сорвало их с орбиты. Это реальный событийный риск.

## 8. Maneuver Suggestion (Рекомендация по маневрированию)

```python
maneuvers = ["Hold Station", "Prograde Boost Recommended", "Inclination Adjustment", "None Required"]
maneuver_suggestion = maneuvers[seed % len(maneuvers)]
```

AI-рекомендация для операторов спутника:

| Рекомендация | Значение |
|---|---|
| **None Required** | Ситуация штатная. Никаких действий не нужно. |
| **Hold Station** | Сохранить текущую позицию на орбите. Возможно, ожидается сближение с другим объектом. |
| **Prograde Boost Recommended** | Рекомендуется включить двигатели в направлении движения, чтобы поднять орбиту и исправить orbital decay. |
| **Inclination Adjustment** | Требуется изменение наклонения орбиты. Это сложный и дорогой манёвр, требующий большого расхода топлива. |

## 11. Риск-скор при детальном анализе (Heuristic Risk Score)

**API Endpoint:** `GET /api/risk/{norad_id}`  
**Источник кода:** `celestial_engine.py → calculate_risk()`

При поиске конкретного спутника по ID/имени система запускает **эвристическую оценку риска** на основе реальных TLE-данных с CelesTrak:

```
Risk Score = 0

▲ +40 баллов → Спутник в LEO (> 11.25 об/сутки)
▲ +30 баллов → Полярная орбита (80° < наклонение < 100°)
▲ +0–20 баллов → Высокий индекс солнарной активности (F10.7)

Максимум: 100 баллов
```

**Уровни риска:**
- `0–40`: 🟢 **LOW** — Стандартная ситуация
- `41–70`: 🟡 **MEDIUM** — Повышенное внимание
- `71–100`: 🔴 **HIGH** — Критический риск

## 12. Вероятность столкновения (Collision Probability, %)

**Источник кода:** `celestial_engine.py → get_satellite_details()`

```python
col_prob = 0.0
if 400 < alt_km < 600: col_prob += uniform(0.5, 2.5)   # Зоны Starlink
if 750 < alt_km < 850: col_prob += uniform(1.0, 3.0)   # Пояс Iridium/Cosmos
if risk == "High":     col_prob += 2.0
```

Три зоны с наивысшей вероятностью столкновения:
- **400–600 км**: Зона мегасозвездий Starlink (~5500 объектов)
- **750–850 км**: Зона, засорённая обломками от столкновения Iridium 33 и Cosmos 2251 (2009)
- **< 400 км**: Зона очень быстрого orbital decay (спутники без тяги быстро сгорают)
